# 腾讯元器 (Yuanqi) 适配说明

## 发布流程

### 1. 准备元数据
已创建 `yuanqi.json` - 元器平台要求的智能体配置文件

### 2. 创建知识库
将以下文档上传到元器知识库：
- `prompts/extract_patterns.md` - 行为提取提示词
- `prompts/synthesize_soul.md` - 人格合成提示词
- `prompts/synthesize_persona.md` - 个人画像提示词

### 3. 配置工作流
在元器平台创建工作流：

```
用户上传文件 → 解析JSON → 分段处理 → Kimi分析 → 合成人格 → 输出报告
```

### 4. 设置工具
需要调用的外部工具：
- **代码执行**: Python 解析脚本
- **大模型**: Kimi K2.5 (通过 API 调用)

### 5. 发布
在 https://yuanqi.tencent.com/ 发布智能体

## 使用方式

用户输入：
```
帮我分析这份聊天记录
[上传 群聊_相亲相爱一家人.json]
```

智能体回复：
```
🎭 伐木累分析大师开始工作...

📊 分析结果：
- 总消息数：841 条
- 时间跨度：2026-02-15 ~ 2026-03-24
- 活跃成员：8 人

👥 成员画像：
• 妈 → 早安打卡机（6:00准时发送🌞）
• 爸 → 转发轰炸机（震惊体专业户）
• 我 → 已读不回（👍维持亲情）
• 三姨 → 养生专家（吃什么致癌/长寿）

📄 详细报告已生成：
- soul.md（群体人格档案）
- persona_妈.md（个人画像）
...
```

## 与 OpenClaw 版本的区别

| 特性 | OpenClaw 版本 | 元器版本 |
|-----|--------------|---------|
| 运行环境 | 本地/自托管 | 腾讯云 |
| API 调用 | 用户自备 Key | 平台统一配置 |
| 文件处理 | 本地 Pipeline | 云端工作流 |
| 输出位置 | 本地文件 | 对话内返回 |
| 定制程度 | 高（可改代码） | 中（配置化） |

## 提示词模板

### 系统提示词
```
你是「伐木累分析大师」，专门分析微信群聊记录的 AI 助手。

你的任务是：
1. 解析用户上传的群聊 JSON 文件
2. 提取每个成员的发言特征和行为模式
3. 识别群体互动模式（谁主导、谁回应、幽默方式等）
4. 生成生动有趣的成员画像

风格要求：
- 用幽默风趣的语言描述
- 适当玩梗（相亲相爱一家人、收到、 etc.）
- 既要有洞察，又要有温度

输出格式：
先用 emoji 总结，再用 markdown 详细说明
```

### 分析提示词
见 `prompts/extract_patterns.md`
